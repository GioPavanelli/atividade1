git init
cria um novo repositório do Git. Por padrão, o git init vai
inicializar a configuração do Git para o caminho do subdiretório .git

git config --global user.name
Serve para fazer login em sua conta virtual digitando seu usuário do GitHub

git config --global user.email
Serve para fazer login em sua conta virtual digitando seu e-mail do GitHub

git remote add origin
Isso é útil quando se deseja iniciar um repositório localmente e, em seguida, é necessário enviá-lo para um repositório remoto mais tarde.

git push
O comando git push é mais comumente usado para postar alterações locais em um repositório central. Depois de modificar o repositório local. Execute comandos push para compartilhar as alterações com membros remotos da equipe.

git commit
o commit é a unidade estrutural do plano de projeto Git.
Eles podem ser vistos como instantâneos ou marcos na linha do tempo de um projeto Git. Eles foram criados usando o comando git commit para capturar o estado do projeto no momento.

git add
O comando git add é famoso por adicionar "adicionar conteúdo" (propondo quaisquer alterações, sejam alterações, adições ou exclusões) de arquivos locais para o índice ou área de teste. Mais tarde, as alterações serão confirmadas por meio do comando git e, finalmente, enviadas via git push para o repositório remoto.

git status
Verifique o status do arquivo. Se houver algum arquivo modificado, adiciona ou excluído. Além disso, se usar uma ramificação no servidor para rastreamento, ele verificará se a versão está adiantada ou atrasada em relação à versão do servidor
